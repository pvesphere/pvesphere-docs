# 集群管理

集群管理功能允许您添加、配置和管理多个 Proxmox VE 集群。

## 添加集群

### 基本信息

- **Cluster Name**: 集群的唯一标识符
- **Name**: 集群的显示名称
- **API URL**: Proxmox VE API 的完整地址（包括协议和端口）

### 认证信息

- **Username**: Proxmox VE API 用户名
- **Password/Token**: API 密码或 Token

### 环境配置

- **Env**: 环境类型（test/dev/prod）
- **Datacenter**: 数据中心名称
- **Region**: 区域标识

## 集群操作

### 启用/禁用集群

- **启用**: 集群将参与调度和监控
- **禁用**: 集群将暂停所有操作，但配置保留

### 测试连接

在添加或编辑集群时，可以点击 "Test Connection" 验证 API 连接。

### 编辑集群

可以随时修改集群配置，包括：

- 更新 API 地址
- 修改认证信息
- 更改环境配置

### 删除集群

删除集群将移除所有相关配置和数据，请谨慎操作。

## 集群状态

### 状态指示

- **Enabled**: 集群已启用且正常运行
- **Disabled**: 集群已禁用
- **Error**: 集群连接失败或出现错误

### 调度状态

- **Schedulable**: 集群可以参与资源调度
- **Non-schedulable**: 集群不参与资源调度

## 最佳实践

1. **命名规范**: 使用清晰的命名规范，便于识别和管理
2. **环境隔离**: 正确设置环境类型，避免误操作
3. **定期检查**: 定期检查集群状态和连接情况
4. **备份配置**: 重要集群配置建议备份
